# Native iOS Device Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a reproducible Capacitor iOS target that builds on this Mac and can be signed, installed, and debugged on the owner's iPhone through Xcode.

**Architecture:** Keep the React/Vite application as the shared implementation and add the standard Capacitor 8.4.1 iOS shell under `ios/`. A `sync:ios` script produces Capacitor-mode web assets and synchronizes plugins; Xcode owns local signing and device installation. The existing Camera plugin provides native camera and photo-library access, while the custom floating screen capture remains Android-only.

**Tech Stack:** React 19, TypeScript 6, Vite 8, Capacitor 8.4.1, Capacitor Camera 8.2, Xcode 26.2, CocoaPods 1.16.2, iOS 15+

## Global Constraints

- Keep `@capacitor/ios` on the Capacitor 8.4.1 version line used by `@capacitor/core`, `@capacitor/android`, and `@capacitor/cli`.
- Preserve bundle identifier `com.brianwong.championsvgc`, app name `Champions VGC Calc`, and web directory `dist` from `capacitor.config.ts`.
- Use the existing `build:mobile` Vite `capacitor` mode; do not add a second iOS-specific web build.
- Commit the generated `ios/` project and permission descriptions, but never commit a Team ID, signing identity, device identifier, certificate, or provisioning profile.
- Support iOS 15 and newer.
- Do not migrate the deprecated Camera `getPhoto` API in this change.
- Do not add an iOS replacement for Android MediaProjection or the floating capture overlay.
- Do not modify files under `android/`.

### Task 1: Add and configure the Capacitor iOS platform

**Files:**
- Modify: `package.json:6-22,63-80`
- Modify: `package-lock.json`
- Create: `ios/` (generated by Capacitor CLI)
- Modify after generation: `ios/App/App/Info.plist`

**Interfaces:**
- Consumes: `capacitor.config.ts` values `appId`, `appName`, and `webDir`; package script `build:mobile`
- Produces: package script `sync:ios`; Xcode workspace `ios/App/App.xcworkspace`; iOS usage-description keys `NSCameraUsageDescription`, `NSPhotoLibraryUsageDescription`, and `NSPhotoLibraryAddUsageDescription`

- [ ] **Step 1: Verify the iOS synchronization entry point is absent**

Run:

```bash
npm run sync:ios
```

Expected: non-zero exit with `Missing script: "sync:ios"`. This is the configuration-level failing acceptance check; the implementation consists of generated and declarative platform files, so no application unit test is added.

- [ ] **Step 2: Install the matching Capacitor iOS runtime**

Run:

```bash
npm install @capacitor/ios@^8.4.1
```

Expected: exit 0; `package.json` gains `"@capacitor/ios": "^8.4.1"` under `dependencies`, and `package-lock.json` records the iOS runtime.

- [ ] **Step 3: Build the web assets required by the native shell**

Run:

```bash
npm run build:mobile
```

Expected: exit 0; TypeScript completes, Vite reports a production build, and `dist/` contains the Capacitor-mode application with base `/`.

- [ ] **Step 4: Generate the standard Capacitor iOS project**

Run:

```bash
npx cap add ios
```

Expected: exit 0; Capacitor creates `ios/App/App.xcworkspace`, copies `dist`, installs native dependencies, and reports the detected Camera plugin.

- [ ] **Step 5: Add the repeatable iOS synchronization script**

Update the scripts block in `package.json` so the mobile scripts are exactly:

```json
"build:mobile": "tsc && vite build --mode capacitor",
"sync:android": "npm run build:mobile && cap sync android",
"sync:ios": "npm run build:mobile && cap sync ios",
"preview": "vite preview"
```

Keep `@capacitor/ios` in `dependencies` beside the other Capacitor runtime packages:

```json
"@capacitor/android": "^8.4.1",
"@capacitor/camera": "^8.2.0",
"@capacitor/core": "^8.4.1",
"@capacitor/ios": "^8.4.1"
```

- [ ] **Step 6: Add iOS camera and photo-library usage descriptions**

Insert the following entries inside the root `<dict>` of `ios/App/App/Info.plist`, immediately before its closing `</dict>`:

```xml

<string>Take photos of game screens to scan teams and battle information.</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>Choose game screenshots to scan teams and battle information.</string>
<key>NSPhotoLibraryAddUsageDescription</key>
<string>Save captured images to your photo library when requested.</string>
```

- [ ] **Step 7: Run the new synchronization acceptance check**

Run:

```bash
npm run sync:ios
```

Expected: exit 0; Vite builds successfully, Capacitor copies the web assets, and synchronization output lists `@capacitor/camera` for iOS.

- [ ] **Step 8: Verify the generated permission configuration**

Run:

```bash

plutil -p ios/App/App/Info.plist
```

Expected: `plutil -lint` reports `OK`; the printed dictionary contains all three usage-description keys with the exact strings from Step 6.

- [ ] **Step 9: Run the automated regression suite**

### Task 2: Install and verify the app on the physical iPhone

**Files:**
- Use: `ios/App/App.xcworkspace`
- May modify locally: `ios/App/App.xcodeproj/project.pbxproj` (Personal Team selection; never commit)
- Repository commits: none; Xcode account and signing state remain local

**Interfaces:**
- Consumes: the compiled `App` scheme and synchronized Camera plugin from Task 1
- Produces: a development-signed installation on the owner's paired iPhone and manual acceptance evidence for launch, camera, photo picker, and iOS platform gating

- [ ] **Step 1: Pair the iPhone with the Mac**

Connect the iPhone by USB, unlock it, accept **Trust This Computer** if prompted, and keep it connected. In iPhone Settings, enable **Privacy & Security → Developer Mode** if Xcode requests it, then restart and confirm Developer Mode when iOS prompts.

Expected: the iPhone appears as an available physical run destination in Xcode.

- [ ] **Step 2: Open the generated workspace**

Run:

```bash
npx cap open ios
```

Expected: Xcode opens `ios/App/App.xcworkspace` with the `App` scheme selected.

- [ ] **Step 3: Configure local automatic signing**

In Xcode, open **Xcode → Settings → Accounts** and add the owner's Apple Account. Select the **App** project and **App** target, open **Signing & Capabilities**, keep **Automatically manage signing** enabled, and choose the owner's Personal Team.

Expected: Xcode reports a valid development signing certificate/profile for bundle identifier `com.brianwong.championsvgc`. Do not commit any signing setting produced by this step.

- [ ] **Step 4: Build and launch on the iPhone**

Choose the connected iPhone as the run destination and press **Run** (`⌘R`).

Expected: Xcode reports a successful build, installs `Champions VGC Calc`, launches it on the iPhone, and attaches the debugger.

- [ ] **Step 5: Verify offline app launch**

With the app already installed, stop the Mac's Vite server if one is running, force-quit the iPhone app, and reopen it from the Home Screen.

Expected: the main calculator UI loads from bundled assets without a development web server.

- [ ] **Step 6: Verify native camera capture**

Open a scan flow, choose **Take Photo**, grant camera permission, take a photo, and confirm the app receives the image and starts its existing scan flow.

Expected: the native iOS camera appears and returns an image; the app does not fall back to Safari or a web URL.

- [ ] **Step 7: Verify native photo-library selection**

Open a scan flow, choose **Choose Photo**, grant or limit photo access, select a screenshot, and confirm the app receives it and starts its existing scan flow.

Expected: the native iOS photo picker appears and returns the selected image.

- [ ] **Step 8: Verify the Android-only control stays unavailable**

Navigate to the calculator screen that exposes one-tap capture on Android.

Expected: the floating/one-tap screen-capture control is absent or disabled on iOS, and the app does not request Android overlay or screen-recording permissions.

- [ ] **Step 9: Confirm the repository remains free of local signing data**

After closing Xcode, run:

```bash
git status --short
```

Expected: either no output, or only `ios/App/App.xcodeproj/project.pbxproj` appears because Xcode stored the selected Personal Team. Keep that signing-only change local and uncommitted so repeat device runs continue to work; exclude it from all later commits.
