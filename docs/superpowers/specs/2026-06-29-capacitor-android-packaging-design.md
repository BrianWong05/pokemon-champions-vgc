## Design (Approach A â€” env-conditional base, keep BrowserRouter)

The only friction is one Vite build serving two runtimes: GitHub Pages (absolute base,
`BrowserRouter`) vs. the Capacitor Android WebView (served from `https://localhost/`). Because
Capacitor's Android `androidScheme: 'https'` serves `dist/` from a real HTTPS origin with a
working History API, `BrowserRouter` works unchanged once the **base is `/`** for the native
build. No router swap (no HashRouter), no app-code changes.

### Build wiring

- `vite.config.ts` â†’ `defineConfig(({ mode }) => ({ base: mode === 'capacitor' ? '/' :
  '/pokemon-champions-vgc/', ...rest }))`. Vitest is unaffected (separate `vitest.config.ts`).
- `package.json` scripts:
  - `"build:mobile": "tsc && vite build --mode capacitor"`
  - `"sync:android": "npm run build:mobile && cap sync android"`
- No source changes: `BrowserRouter basename={BASE_URL}`, `getDb`, and both image components
  all read `BASE_URL`, which becomes `/` in the native build automatically.

### Capacitor config

- Deps: `@capacitor/core`, `@capacitor/android` (runtime) + `@capacitor/cli` (dev), latest.
- `capacitor.config.ts`: `appId: 'com.brianwong.championsvgc'`, `appName: 'Champions VGC Calc'`,
  `webDir: 'dist'`, default `androidScheme: 'https'`.
- `npx cap add android` generates the `android/` Gradle project; `npx cap sync android` copies
  the mobile `dist/` (incl. `vgc_pokemon.db` and the wasm) into the app's web assets.

### Committed vs ignored

- Commit: `capacitor.config.ts`, the `vite.config.ts` / `package.json` / lockfile changes, and
  the generated **`android/`** project (slice #4's native overlay plugin will live there).
- Ignored: Capacitor writes `android/.gitignore` for build artifacts (`build/`, `.gradle`,
  `local.properties`, etc.). Add `android/app/src/main/assets/public/` if `cap` doesn't ignore
  the synced web bundle (it is regenerated by `cap sync`, not source).

## Risks / open questions

- **JDK 25 vs Android Gradle Plugin.** AGP may require JDK 17/21. Mitigation: run Gradle with
  Android Studio's bundled JBR (`org.gradle.java.home` or `JAVA_HOME`).
- **AVD availability.** If no emulator/system image exists, creating one may be slow; if the
  emulator genuinely can't run in this environment, fall back to a verified APK build + report
  (and flag that criterion 3's launch step is unverified).
- **Offline missing-image fallback.** `PokemonImage`'s `onError` points to a remote GitHub raw
  URL, so a *missing* thumbnail won't load offline. Pre-existing, minor, out of scope.
